# Phase 4: 측정 파이프라인

## 목표

비용/품질 데이터를 정량적으로 추적하여 데이터 기반 개선 루프를 구축.
"측정하지 않으면 개선할 수 없다."

## 배경

### 현재 상태

모든 LLM 기반 개발 프레임워크가 공유하는 문제: "효과가 있는가?"에 대한 정량적 답이 없다.

- claude-config: "formal benchmarks를 돌려보지 않았다"
- SuperClaude: 기능 목록은 있지만 효과 측정 없음
- ECC: 50K 스타는 인기 지표이지 품질 지표가 아님

### 측정이 필요한 질문들

1. QR 게이트를 추가하면 토큰이 얼마나 더 소비되는가?
2. QR 게이트가 재작업을 얼마나 줄이는가?
3. 도메인 프로필이 리뷰 정확도에 영향을 미치는가?
4. 컨텍스트 관리 레이어가 토큰 소비를 실제로 줄이는가?
5. Micro/Standard/Full 중 어떤 워크플로우가 어떤 작업에 최적인가?

## 수집 방법

### 옵션 1: ccusage 연동 (기존 도구)

```bash
# 세션별 토큰 사용량 조회
ccusage --session --format json

# 일별 비용 분석
ccusage --daily --model-breakdown
```

장점: 이미 존재하는 도구, 설치 간단
단점: QR 통과율 등 커스텀 메트릭은 별도 수집 필요

### 옵션 2: cc-history JSONL 파서 (커스텀)

Claude Code 대화 기록(`~/.claude/projects/*/`)을 직접 분석.

수집 가능 항목:
- 스킬 호출 패턴 (`python3 -m skills\.([a-z_]+)\.`)
- 모델별 토큰 소비 (message.usage)
- 서브에이전트 상관 관계 (parentUuid)
- QR 반복 감지 (같은 파일에 대한 연속 수정)

### 옵션 3: 세션 로그 (최소 구현)

각 세션 종료 시 수동으로 기록:

```json
{
  "session_id": "2026-03-10-auth-feature",
  "workflow": "standard",
  "domain_profile": "fintech",
  "milestones": 3,
  "files_changed": 7,
  "qr_iterations": [1, 2, 1],
  "total_tokens": 150000,
  "notes": "M2에서 Temporal 위반으로 QR 재시도"
}
```
