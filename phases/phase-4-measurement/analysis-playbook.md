# Analysis Playbook

10개 이상의 세션 데이터가 축적된 후 수행할 분석 절차.

---

## 분석 1: 워크플로우 Tier 비교

### 질문

"Micro, Standard, Full 중 어떤 워크플로우가 어떤 복잡도에서 최적인가?"

### 데이터 요구

- Micro 세션 3개 이상
- Standard 세션 3개 이상
- Full 세션 2개 이상

### 분석 절차

1. Tier별 그룹핑
2. 각 그룹의 평균 계산:
   - 토큰/파일 = total_tokens / files_changed
   - 재작업률 = rework_count / files_changed
3. 비교 테이블 작성

```
                Micro    Standard    Full
토큰/파일:       [?]      [?]        [?]
재작업률:        [?]      [?]        [?]
QR 통과율:       N/A      [?]        [?]
```

### 예상 결과

- Micro: 토큰/파일 최저, 재작업률 최고 (검증 없으므로)
- Full: 토큰/파일 최고, 재작업률 최저 (검증 비용이 재작업 감소로 상쇄)
- Standard: 중간

### 판단 기준

- Full의 토큰/파일이 Micro의 2배 이내이면서 재작업률이 50% 이상 낮으면:
  → Full의 비용 대비 효과가 정당화됨
- Full의 토큰/파일이 Micro의 3배 이상이면서 재작업률 차이가 20% 미만이면:
  → Full이 과도한 오버헤드, Standard로 충분

## 분석 2: QR 게이트 효과

### 질문

"QR 게이트가 추가하는 토큰 비용은 재작업 절감으로 상쇄되는가?"

### 데이터 요구

- Standard 또는 Full 세션 5개 이상 (QR이 있는 세션)
- Micro 세션 5개 이상 (QR이 없는 세션, 비교군)

### 분석 절차

1. QR 비용 추정:
   - Standard/Full 세션의 평균 토큰에서 Micro 세션의 토큰/파일 × 파일 수를 뺌
   - 차이 = QR 오버헤드 추정치
2. 재작업 절감 추정:
   - Micro의 재작업률 - Standard/Full의 재작업률
   - 절감된 재작업 × 평균 재작업 토큰 비용
3. 비용-편익 비교

### 판단 기준

- QR 비용 < 재작업 절감 → QR 게이트 유지
- QR 비용 > 재작업 절감 × 2 → QR 간소화 검토
- 데이터 불충분 → 추가 수집

## 분석 3: 도메인 프로필 효과

### 질문

"도메인 프로필이 리뷰 정확도와 비용에 영향을 미치는가?"

### 데이터 요구

- 같은 도메인의 세션 3개 이상 (프로필 적용)
- 같은 도메인의 세션 3개 이상 (프로필 미적용, 비교군)

### 분석 절차

1. 프로필 적용 그룹 vs 미적용 그룹 비교
2. 비교 항목:
   - MUST 발견 건수 (프로필이 도메인 특화 MUST를 얼마나 잡는가)
   - QR 반복 횟수 (프로필이 첫 시도 통과율을 높이는가)
   - 토큰 소비 (프로필이 토큰 오버헤드를 추가하는가)

### 판단 기준

- 프로필 적용 시 MUST 발견이 증가하고, QR 반복은 감소하면:
  → 프로필이 리뷰 품질을 향상시킴
- 프로필 적용 시 토큰이 10% 이상 증가하지만, 재작업은 동일이면:
  → 프로필 비용 대비 효과 미미, 간소화 필요

## 분석 4: 컨텍스트 관리 효과

### 질문

"델타 로그/롤링 요약이 토큰 소비를 줄이고 품질을 유지하는가?"

### 데이터 요구

- Full 워크플로우 세션 (5+ 마일스톤) 2개 이상
  - 델타 로그 적용 1개
  - 델타 로그 미적용 1개

### 분석 절차

1. 마일스톤별 토큰 소비 추이 비교:
   - 미적용: 마일스톤 N의 토큰 = f(N)으로 증가하는가?
   - 적용: 마일스톤 N의 토큰이 일정 범위 내 유지되는가?
2. 정보 손실 확인:
   - 후반 마일스톤에서 초반 결정을 참조해야 할 때 성공률

### 판단 기준

- 미적용 시 토큰이 마일스톤당 20% 이상 증가하고,
  적용 시 10% 이내로 유지되면: → 컨텍스트 관리 효과 입증
- 정보 손실이 발생하면: → 델타 엔트리 스키마 보강 필요

---

## 분석 일정

```
세션 5개 축적  → 분석 1 (워크플로우 비교) 가능
세션 10개 축적 → 분석 2 (QR 효과) 가능
프로필 적용 후 → 분석 3 (도메인 프로필) 가능
Full 워크플로우 → 분석 4 (컨텍스트 관리) 가능
```
