# 시스템 아키텍처

## 설계 원칙

1. **기존 환경을 교체하지 않고 확장한다** — SuperClaude 기반 유지, 부족한 레이어 추가
2. **원칙과 구현을 분리한다** — 원칙은 도구 독립적, 구현은 도구별 어댑터
3. **문서가 코드에 선행한다** — 규칙과 기준을 먼저 정의, 자동화는 그 다음
4. **측정 없이 최적화하지 않는다** — 데이터 기반 개선 루프

## 레이어별 상세

### Layer 1: 프로세스 모델 (Phase 1)

현재 환경에 프로세스 강제력을 추가한다.

```
작업 요청
    │
    ▼
[복잡도 판정]
    │
    ├─ Micro (1-2 파일) ──→ Developer 직접 실행
    │
    ├─ Standard (3-10 파일) ──→ 계획 → 실행 → QR 1회
    │
    └─ Full (10+ 파일) ──→ 계획 → [QR 게이트] → 실행 → [QR 게이트] → 문서화
                                                   │
                                              마일스톤별 반복
```

**QR 게이트 상세**:
```
[QR 실행]
    │
    ├─ PASS ──→ 다음 단계
    │
    └─ FAIL
        │
        ├─ iter 1-2: MUST + SHOULD + COULD 전부 차단
        ├─ iter 3-4: MUST + SHOULD만 차단
        └─ iter 5+:  MUST만 차단 (무한 루프 방지)
```

### Layer 2: 도메인 프로필 (Phase 2)

프로젝트별 에이전트 행동 커스터마이징.

```
┌────────────────────────┐
│ 도메인 프로필 (YAML)     │
├────────────────────────┤
│ 우선순위 매트릭스        │ ──→ QR 심각도 가중치 조정
│ 규약 오버라이드          │ ──→ 테스트/리뷰/문서화 기준 변경
│ 도메인 지식             │ ──→ 금지/필수 패턴 주입
│ 워크플로우 커스터마이징   │ ──→ 자동화 수준 조정
└────────────────────────┘
```

**컨벤션 계층에서의 위치**:
```
Tier 1:   사용자 명시 지시 (가장 높은 우선순위)
Tier 1.5: 도메인 프로필
Tier 2:   프로젝트 문서 (CLAUDE.md, README.md)
Tier 3:   기본 규약
```

### Layer 3: 컨텍스트 관리 (Phase 3)

토큰 효율적 누적 지식 관리.

```
[마일스톤 N 완료]
    │
    ▼
[델타 생성] ── "무엇이 변경됐고, 어떤 결정을 내렸는가"
    │
    ▼
[결정론적 병합] ── LLM 호출 없이 구조적 병합
    │
    ▼
[누적 컨텍스트] ── 구조화된 불릿 포인트
    │
    ▼
[마일스톤 N+1 시작] ── 누적 컨텍스트 + 현재 작업 컨텍스트만 주입
```

**구조적 섹션** (항상 유지):
```
## 수정된 파일
- src/auth.py: JWT 검증 로직 추가
- src/models/user.py: role 필드 추가

## 내린 결정
- JWT > 세션: 무상태 우선, 수평 확장 고려
- bcrypt > argon2: 기존 의존성 호환

## 미해결 사항
- 토큰 갱신 전략 미확정

## 다음 단계
- 토큰 갱신 엔드포인트 구현
```

### Layer 4: 측정 (Phase 4)

데이터 기반 개선 루프.

```
[작업 세션]
    │
    ▼
[메트릭 수집]
    │ 토큰 소비, QR 통과율, 재작업 횟수
    ▼
[분석]
    │ 워크플로우별, 도메인별, 단계별 비교
    ▼
[개선 방향 도출]
    │ "QR iter 3+ 발생 원인의 70%가 Temporal 위반"
    ▼
[프로세스/프로필 조정]
```

## 출력물 흐름

각 Phase가 생산하는 산출물과 소비 관계:

```
Phase 1 산출물:
  ├── process-model.md (프로세스 규칙)
  ├── qr-rules.md (QR 3-Tier 규칙)
  ├── temporal-rules.md (Temporal 규칙)
  └── convention-tiers.md (컨벤션 계층)
        │
        ▼ Phase 2가 참조
Phase 2 산출물:
  ├── schema.yaml (프로필 스키마)
  ├── profiles/fintech.yaml
  ├── profiles/game-dev.yaml
  └── profiles/startup-mvp.yaml
        │
        ▼ Phase 3가 참조 (도메인별 컨텍스트 우선순위)
Phase 3 산출물:
  ├── delta-schema.json (델타 엔트리 스키마)
  ├── merge-rules.md (결정론적 병합 규칙)
  └── summary-template.md (롤링 요약 템플릿)
        │
        ▼ Phase 4가 측정
Phase 4 산출물:
  ├── metrics-definition.md (메트릭 정의)
  ├── collection-scripts/ (수집 도구)
  └── reports/ (분석 결과)
```
