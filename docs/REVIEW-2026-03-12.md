## 3. 코드 리뷰 (tools/)

### 3.1 CRITICAL

#### C1. Gmail/GitHub 핸들러가 항상 NotImplementedError 발생

**파일**: `tools/src/gmail/tools.py` (lines 137-161), `tools/src/github/tools.py` (lines 230-236)

`_make_*_handler` 팩토리 함수들이 `GmailClient`/`GitHubClient` 인스턴스를 연결하지 않고, 무조건 `NotImplementedError`를 raise하는 스텁을 반환한다. MCP 도구로 등록은 되지만 호출하면 항상 실패한다.

```python
# gmail/tools.py — 핸들러 팩토리가 스텁만 반환
def _make_list_handler(server):
    async def handler(**kwargs):
        raise NotImplementedError("Gmail client not configured")
    return handler
```

`GmailClient`와 `GitHubClient` 클래스는 구현되어 있으나, 도구 등록 시 이 클래스들의 인스턴스를 생성하고 핸들러에 주입하는 로직이 없다.

**영향**: LLM이 `gmail_list_messages`, `github_list_issues` 등을 호출하면 항상 `{"success": false, "error": "...NotImplementedError..."}` 응답.

**조치**: 환경변수에서 토큰을 로드하여 클라이언트를 생성하고, 핸들러에 주입하는 초기화 로직 구현 필요.

#### C2. 토큰 sanitization 패턴이 불완전

**파일**: `github/tools.py:14`, `github/actions/ci_environment.py:12`, `github/actions/git_utils.py:14`

현재 패턴: `ghp_[A-Za-z0-9]+|gho_[A-Za-z0-9]+|Bearer\s+\S+`

누락된 GitHub 토큰 형식:
- `github_pat_` — fine-grained personal access token
- `ghs_` — server-to-server token (GitHub App)
- `ghr_` — refresh token
- `gha_` — Actions token

이 형식의 토큰이 에러 메시지나 CI 로그에 포함되면 sanitize되지 않고 LLM 응답에 노출될 수 있다.

**조치**: 통합 패턴을 `shared/sanitize.py`에 정의하고 모든 모듈에서 import.

#### C3. TaskManager 인스턴스 간 ID 충돌 가능

**파일**: `tools/src/shared/task/manager.py:93-108`

`_counter`가 인스턴스 변수이므로, 두 개의 `TaskManager`가 동시에 존재하면 같은 `TASK-001`을 생성할 수 있다. 현재는 `get_server()` 싱글톤으로 보호되지만, 테스트나 향후 확장에서 문제 가능.

**조치**: (a) 카운터를 파일 기반으로 영속화하거나 (b) ID에 source prefix를 포함하여 유일성 보장.

### 3.2 MAJOR

#### M1. 매 API 호출마다 새 httpx.AsyncClient 생성

**파일**: `gmail/client.py:70`, `github/client.py:85,146`

```python
async with httpx.AsyncClient() as client:  # 매번 새 연결
    response = await client.request(...)
```

HTTP keep-alive를 활용하지 못하고, 매 요청마다 TCP 핸드셰이크 + TLS 협상이 반복된다. `list_messages`로 50개 메시지 조회 시 51개의 독립 TCP 연결이 생성된다.

**조치**: 클라이언트를 클래스 `__init__`에서 생성하고, `async with` 패턴 대신 명시적 `aclose()` 관리.

#### M2. OAuth 콜백에 타임아웃 없음

**파일**: `tools/src/shared/auth/oauth_flow.py:153-188`

사용자가 OAuth 인증을 완료하지 않으면 `await code_future`가 영구적으로 block된다. 서버 프로세스가 종료될 때까지 해제되지 않는다.

**조치**: `asyncio.wait_for(code_future, timeout=300)` 같은 타임아웃 래핑.

#### M3. OAuth error 파라미터 미처리

**파일**: `tools/src/shared/auth/oauth_flow.py:158-171`

OAuth 제공자가 사용자 거부 시 `?error=access_denied`로 리다이렉트하는데, 핸들러가 `code` 파라미터만 확인한다. 거부 시 `code_future`가 영원히 resolve되지 않아 M2의 hang과 결합된다.

**조치**: `error` 파라미터 확인 후 `code_future.set_exception()`으로 에러 전파.

#### M4. _get_token_url이 string 리터럴 키 사용

**파일**: `tools/src/shared/auth/oauth_flow.py:145-151`

```python
urls = {
    "gmail": "https://oauth2.googleapis.com/token",  # str 리터럴
    "github": "https://github.com/login/oauth/access_token",
}
return urls.get(self._config.service)  # ToolSource enum
```

`ToolSource`가 `StrEnum`이므로 현재 동작하지만, 새 소스를 추가하면 조용히 `None`을 반환하여 refresh가 실패한다.

**조치**: 키를 `ToolSource.GMAIL`, `ToolSource.GITHUB`으로 변경.

#### M5. TaskManager가 FileTaskStore에 직접 의존

**파일**: `tools/src/shared/task/manager.py:16`

```python
def __init__(self, store: FileTaskStore | None = None) -> None:
```

인터페이스(Protocol)가 아닌 구체 클래스에 의존하여 DIP 위반. 인메모리 스토어로 테스트하려면 FileTaskStore를 상속해야 한다.

**조치**: `TaskStoreProtocol`을 정의하고 타입 힌트를 변경.

#### M6. update_status가 O(n*m) 풀스캔

**파일**: `tools/src/shared/task/manager.py:73-91`

모든 소스 파일을 로드한 후 모든 태스크를 순회하여 ID를 찾는다. 소스가 2개, 태스크가 100개이면 200회 비교. 규모가 커지면 비효율적.

**조치**: task_id에 source prefix를 포함하거나, ID→source 매핑 인덱스를 유지.

#### M7. Gmail list_messages가 순차 fetch

**파일**: `tools/src/gmail/client.py:40-44`

```python
for msg_id in message_ids:
    detail = await self._request(...)  # 순차 대기
```

N개 메시지를 1개씩 순차 요청. `asyncio.gather`로 병렬화하면 N배 빨라진다.
